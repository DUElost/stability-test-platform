选机工作台 + 发起驾驶舱

把「四步表单向导」收成更贴工程师心智的流程：Plan → 选机 → 发起。 第 2 步默认矩阵主舞台（≈30px 格），表格为辅；发起前用全宽驾驶舱替代瘦 Dialog。 下列四页可点交互原型，用于评审信息架构与视觉语言——非最终像素稿。

按节点槽位表 · 历史墙钟参考 · 疑似重复黄警 · 参数 ⓘ 解释。

本轮建议验收关注点

  • 矩阵是否应默认开启（表为辅），还是双视图平权？
  • 向导是否接受压成 3 态（合并原「数量与版本确认」到顶栏实时 chip）？
  • 色板：就绪 / 阻塞(斜纹) / 占用 / 离线 + 选中描边——色弱可读性是否够？
  • 发起驾驶舱：只做「超槽位将排队」+ 整次耗时参考，不算精确开跑时刻——文案是否接受？
  • Minimap：点格 = 定位高亮；hover ✕ 才移除——是否符合直觉？
