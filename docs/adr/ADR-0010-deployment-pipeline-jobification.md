# ADR-0010: 部署能力作业化（异步、幂等、可回放）
- 状态：Proposed
- 优先级：P1
- 目标里程碑：M2
- 日期：2026-02-18
- 决策者：平台研发组
- 标签：部署, 异步作业, 幂等, 重构

## 背景

当前部署功能已可用，但仍存在演进空间：

- 同步 API 执行时间长，接口阻塞风险高。
- 步骤日志缺少标准化结构，重试/回放语义弱。
- SSH 执行与文件上传路径混用，作业边界不清晰。

## 决策

将部署能力重构为“部署作业流水线”：

- API 只负责创建作业并返回 `deployment_id`。
- 后台 worker 执行分阶段步骤：连接、准备目录、同步、安装、启动、验收。
- 每个步骤记录结构化状态、耗时、错误码，可重试且幂等。
- 日志支持增量拉取或事件推送，前端可实时展示。

## 备选方案与权衡

- 方案 A：继续沿用同步流程 + 少量补丁。
  - 优点：短期投入低。
  - 缺点：稳定性和可运维性提升有限。
- 方案 B：作业化重构（当前提案）。
  - 优点：可观测、可重试、可扩展到批量部署。
  - 缺点：实现成本高，需要引入更明确的作业状态机。

## 影响

- 正向影响：提升批量部署可靠性和可追踪性。
- 代价：部署模块会从“脚本调用”演进为“作业系统”，需新增测试与回归基线。

## 落地与后续动作

- 第一步：定义部署步骤状态机与错误码字典。
- 第二步：改造 `deploy` API 为“创建作业 + 查询作业”模式。
- 第三步：补齐失败重试、取消、幂等键支持。
